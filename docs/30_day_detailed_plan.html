Week 1
Docker Scaffolding · Days 1–7
Day 1
AllTeam kickoff — agree on branch strategy: main / dev / feature/*. Install Docker Desktop on all machines.
ACreate GitHub repo, initialise README, configure branch protection rules on main (require PR reviews).
Day 2
AWrite docker-compose.yml — add postgres, redis, rabbitmq, minio containers. Configure named volumes and shared network.
CWrite .env.example with all required variables (DB credentials, JWT secret, MinIO keys). Add inline comments for each.
Day 3
AConfigure Makefile targets: make up, make down, make logs. Verify all 4 infra containers start and become healthy.
BScaffold FastAPI gateway project. Add GET /health returning {"status":"ok","version":"0.1.0"}. Add requirements.txt.
Day 4
CScaffold Next.js 14 (App Router). Create folder structure: app/, components/, lib/, public/. Install Tailwind CSS.
BAdd FastAPI gateway service to docker-compose.yml. Verify it connects to postgres and redis on startup.
Day 5
AResolve Docker networking issues — WSL2 port conflicts especially. Confirm all 5 containers (infra + gateway) are simultaneously healthy.
CSet up GitHub Actions CI: Python linting with ruff, JS linting with eslint. Push first passing CI run.
Allocate 4h buffer for WSL2 port conflicts — common first-week blocker
Day 6
CAdd Next.js to docker-compose.yml. Build placeholder homepage with project name and navigation shell.
BVerify FastAPI /health is reachable from Next.js (inside Docker network). Configure CORS headers to allow http://localhost:3000.
Day 7
AllIntegration day: run make up. Verify 6 containers green. Hit /health → 200. Confirm Next.js loads in browser.
AllCommit all files. Tag release v0.1.0-infra-scaffold. Push to GitHub.
Milestone: All infra containers healthy · /health returns 200 · Next.js placeholder live
Week 2
Database Schema · Days 8–14
Day 8
AWrite SQLAlchemy ORM models for core tables: users, sessions, jobs. Define columns, types, and foreign keys.
BWrite shared Pydantic DTOs (request/response schemas). Init shared/ Python package with pyproject.toml for editable install.
Day 9
AContinue ORM models: detection_results, text_results, image_results, audio_results, reports. Add all indexes.
CConfigure editable install (pip install -e ./shared). Verify both FastAPI gateway and future services can import shared cleanly.
Day 10
AWrite Alembic migration files 001–004. Add IF NOT EXISTS guards for all CREATE TYPE statements to prevent enum conflicts on re-runs.
BPeer-review ORM models with Student A. Complete remaining Pydantic DTOs to cover all 8 tables.
Day 11
AWrite Alembic migration files 005–008. Run alembic upgrade head. Verify all indexes created via \d+ table_name in psql.
CAdd GitHub Actions step: run alembic check on every PR. CI must fail if models and migrations are out of sync.
Day 12
ATest full alembic downgrade base then alembic upgrade head cycle. Confirm all 8 migrations are idempotent and reversible.
BWrite scripts/seed_db.py — inserts 2 test users, 5 sample jobs, sample results for local dev and testing.
Day 13
CAdd GET /health/db endpoint to FastAPI — executes a lightweight SELECT 1 against postgres and returns connection status.
BWrite docs/database-schema.md — document all 8 tables with field names, types, constraints and entity relationships.
Day 14
AllReview all 8 tables together. Open a test PR with a dummy model change — confirm CI correctly blocks it on alembic check failure.
AllMerge to dev. Tag release v0.2.0-db-schema.
Milestone: All 8 tables verified · alembic current shows head · CI blocks migration drift
Week 3
Auth Service & JWT Flow · Days 15–21
Day 15
ABuild auth-service FastAPI app. Implement POST /auth/register: validate email uniqueness, hash password with bcrypt, insert user row.
CBuild Next.js register page: form with name, email, password fields, client-side validation, and error messages.
Day 16
AImplement POST /auth/login: verify bcrypt hash, generate JWT access token (15min expiry) + refresh token (7 days). Store refresh token in Redis.
BWrite JWT utility module: create_token(), verify_token(), extract_claims(). Unit-test each function independently.
Day 17
AImplement POST /auth/refresh: validate refresh token key in Redis, issue new access token. Implement POST /auth/logout: delete refresh key from Redis.
CBuild Next.js login page. Implement token storage in httpOnly cookie on response from login API. Handle login errors (401, 422).
Day 18
BIntegrate auth into API Gateway: add Depends(get_current_user) middleware. Create GET /api/v1/auth/me protected endpoint returning current user profile.
CFix CORS between Next.js (port 3000) and FastAPI (port 8000). Set allow_credentials=True, configure allowed origins precisely.
CORS with httpOnly cookies is tricky — allocate 4h buffer for this day
Day 19
AWrite 12 pytest unit tests covering: register, login, failed login (wrong password), refresh, logout, access protected route, expired token → 401, missing token → 401.
CBuild protected route middleware in Next.js: check for JWT cookie, redirect unauthenticated users to /login. Wrap dashboard pages.
Day 20
BAdd auth-service to docker-compose.yml. Verify gateway can call auth-service using Docker internal DNS. Test full service-to-service auth flow.
CAdd pytest auth tests to GitHub Actions CI. All 12 tests must pass on every PR before merge is allowed.
Day 21
AllFull end-to-end manual test: register → login → call /api/v1/auth/me → logout → confirm next request returns 401.
AllConfirm all 12 unit tests pass in CI. Merge to dev. Tag v0.3.0-auth.
Milestone: Complete auth flow working · 12 tests passing · JWT + httpOnly cookies
Week 4
MinIO Storage & RabbitMQ Queues · Days 22–30
Day 22
AImplement StorageService class: upload_file() using MinIO put_object, download_file() using get_object, generate_presigned_url().
CBuild Next.js UploadDropzone component using react-dropzone. Support drag-and-drop and click-to-browse. Show file name and size preview.
Day 23
AImplement QueueService class: publish(queue_name, payload) method publishing JSON messages to RabbitMQ. Test with text_jobs, image_jobs, audio_jobs queues.
BAdd python-magic MIME validation to Dockerfiles: install libmagic1 system package. Without this, MIME detection silently returns wrong types.
Day 24
AWrite POST /api/v1/detect/image: validate MIME → upload to MinIO → create job row in DB (status=queued) → publish to image_jobs queue → return 202 + job_id.
CAfter upload, display job ID badge and status chip (Queued) in the UI. Show loading spinner while awaiting first status poll.
Day 25
BImplement GET /api/v1/jobs/{id}: return job status, created_at, updated_at, and result_id if complete. Return 404 for unknown job IDs.
CBuild JobStatusCard component: polls /jobs/{id} every 3 seconds. Renders state machine: Queued → Processing → Complete / Failed. Stops polling on terminal state.
Day 26
AWrite stub RabbitMQ consumer: reads from image_jobs queue, logs message payload, waits 2 seconds, then updates job status to complete in DB.
BAdd MinIO bucket auto-creation on startup: ensure buckets uploads and reports exist before any service accepts requests. Add to startup event handler.
Day 27
CWrite POST /api/v1/detect/text and POST /api/v1/detect/audio stub endpoints — same pattern as image. Wire into frontend upload flow.
BWrite integration test: upload test JPEG → assert MinIO object exists → assert DB job row exists (status=queued) → assert RabbitMQ image_jobs queue has 1 message.
Day 28
ADebug edge cases: file size limits (add 10MB cap), upload timeouts, RabbitMQ message serialisation issues. Add 413 / 415 / 503 error responses.
BAdd comprehensive error handling to the detect endpoints: wrong MIME type (415), file too large (413), queue unavailable (503 with retry-after header).
CAdd upload progress bar to UploadDropzone. Handle upload failure state — show retry button with error message.
Day 29
AllFull system test: upload a text file, image, and audio stub from the Next.js frontend. Verify all 3 queues receive messages and all 3 jobs transition to complete in the UI.
AllFix any remaining bugs. Update README setup guide. Run alembic check — confirm no migration drift.
Day 30
AllLive demo: upload file from Next.js → JobStatusCard polls → status flips to Complete → confirm object in MinIO, job row in DB, and queue message consumed by stub.
AllMonth 1 retrospective. Document blockers and lessons learned. Tag release v0.4.0-month1-complete. Prep for Month 2: Text Modality & RAID dataset.
Milestone: Full upload pipeline · Job status polling in UI · All 3 queue stubs operational
